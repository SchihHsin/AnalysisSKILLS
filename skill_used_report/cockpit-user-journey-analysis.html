Competitive Analysis · 用户旅程专题

无线网络智能体与鸿蒙智行智能座舱的跨域用户旅程对标

本报告把“车主在智能座舱中完成出行任务”的旅程，映射到“运维人员在无线网络智能体中完成网络健康闭环”的旅程。两者场景不同，但核心机制相似：用户表达目标，系统理解并持续执行，关键风险出现时让人接管，最终沉淀为可解释的结果。

鸿蒙智行智能座舱 以小艺、车机地图、座舱提醒、车控与 ADS 辅助驾驶为核心的出行协作界面。
无线网络智能体 以意图输入、指标监控、GIS 态势、事件接管和日报复盘为核心的运维协作界面。
自然意图入口“帮我把重点区域健康度稳定到 98 分”对应座舱里的“带我去最近的充电站”。
风险接管提示陆家嘴容量均衡策略需人工确认，对应座舱中驾驶员随时接管的责任边界。
01 · 核心结论

它们不是同品类竞品，但共享同一种“智能副驾”产品范式

A

相似的起点：用户不再点菜单，而是表达目标

鸿蒙座舱通过小艺承接导航、车控、娱乐、问答等自然语言任务；无线智能体应承接“网络健康度、弱覆盖、质差、投诉、节能”等业务目标。

B

相似的过程：系统持续感知环境变化

座舱感知路况、座舱状态、驾驶风险；无线智能体感知 KPI/KQI、投诉、工单、站点和区域态势。差异只在“路况”变成了“网络状态”。

C

相似的责任：自动化必须保留人工接管

ADS 是辅助驾驶，驾驶员仍需保持注意；无线智能体也只能自动处理低风险动作，高风险策略、派单和外部干扰仍需人确认。

02 · 双泳道用户旅程图

从“出行副驾”到“网络运维副驾”的旅程映射

角色 / 阶段
1. 启动任务
2. 表达意图
3. 系统执行
4. 风险提醒
5. 人工接管
6. 复盘沉淀
鸿蒙座舱用户
上车 / 唤醒座舱车主进入车内，座舱加载账号、车辆状态、导航和娱乐环境。
对小艺说目标例如找目的地、规划路线、调节车控、处理复杂问答。
导航与车控执行系统结合地图、车况和用户指令执行，并在屏幕/语音上反馈。
主动提醒安全、天气、胎压、座舱环境、驾驶辅助边界等被主动提示。
驾驶员干预辅助驾驶不能替代驾驶员，用户需要随时关注并接管。
行程完成目的地到达、能耗/里程/导航历史沉淀为下一次服务上下文。
无线运维用户
打开驾驶舱运维人员进入 PC 页面，看到 4 个意图、指标卡、GIS 态势和事件流。
输入网络目标例如“上海重点区域健康度 98 分”“降低投诉热点”。
智能体监控处置系统拆解区域、指标、阈值和策略，持续监控并执行低风险动作。
事件卡 + GIS 气泡新异常进入左侧事件流，同时在上海 GIS 上出现定位气泡。
人工接管确认高风险参数、派单、外部干扰、容量保障需要运维人员确认。
日报邮件复盘智能体汇总今天做了什么、网络提升多少、还有哪些待跟进。
关键触点
个性化上下文座舱账号 / 运维角色、区域权限、日报订阅。
自然语言意图从命令式操作升级为目标式表达。
过程透明系统必须告诉用户正在做什么、为什么这样做。
主动唤醒从“用户找问题”变成“系统把问题带到用户面前”。
责任边界可自动 / 需确认 / 禁止自动要清晰分层。
证据链结果必须可解释、可追溯、可汇报。
摩擦与机会
摩擦：入口像监控台
机会：做成任务驾驶舱
摩擦：意图可能模糊
机会：自动生成可编辑意图胶囊
摩擦：黑盒自动化不可信
机会：展示思考态与依据
摩擦：告警疲劳
机会：只推需决策事件
摩擦：责任不清
机会：接管原因 + 风险提示
摩擦：日报容易流水账
机会：按目标呈现成效
03 · 触点对比表

重点比较鸿蒙智行与本产品的相似功能点

UX 触点 鸿蒙智行 / 鸿蒙座舱车主出行场景 无线网络智能体运维闭环场景 对本产品的启发
意图入口 小艺承接导航、车控、娱乐、问答等自然语言任务，且新一代座舱强调聊天式跨域交互。 运维人员输入“健康度 98 分”“降低投诉热点”等目标，智能体拆解为指标、区域、策略和周期。 做“意图胶囊”
把自然语言自动结构化为区域、目标值、指标、截止时间，并允许人工编辑。
智能体思考态 车载小艺可区分快思考和慢思考，复杂任务会出现思考中样式，再输出结果。 当前产品需要告诉用户智能体正在关联哪些网络证据，如 MR、KQI、投诉、工单、参数历史。 展示推理进度
例如“正在定位质差根因：上行干扰 62% / 容量拥塞 28%”。
态势地图 座舱导航把路线、环境、驾驶辅助状态整合为驾驶员的主视图。 GIS 应整合指标热力、站点扇区、事件气泡和成效图层，是运维人员的主视图。 地图要成为任务画布
指标卡切换图层，事件卡定位风险，日报回放今日处置轨迹。
主动提醒 座舱可围绕安全、舒适、环境和车控状态给出主动提醒或建议。 左侧待接管事件与右侧 GIS 气泡就是主动提醒的运维版本。 少而准
只推“需要人决策”的事件，避免把页面变成传统告警列表。
接管机制 ADS 明确是辅助驾驶，驾驶员需要保持注意并随时干预。 高风险参数、派单、跨区域影响、用户投诉升级等必须人工接管。 接管原因必须显性
每条事件标明“为什么不能自动处理”“接管后推荐动作”“不处理后果”。
多屏 / 多区域协同 鸿蒙座舱强调多媒体多屏流转和座舱应用生态。 PC 驾驶舱需要左侧意图与事件、右侧 GIS、日报邮件之间的信息流转。 做跨模块联动
点击意图切换默认地图图层；点击事件定位气泡；点击日报条目回看证据。
安全与权限 鸿蒙智行强调数据安全、权限提醒、敏感权限记录和多用户隔离。 网络运维涉及位置栅格、投诉、工单和策略操作，更需要权限透明与操作留痕。 引入“数据来源与权限标签”
日报和事件详情显示数据来源、脱敏状态、操作者与审批链。
04 · 推荐方向

将无线网络智能体定位为“网络运维智能座舱”

不要把它定义成“指标看板 + 告警列表”。它更接近智能座舱：用户给目标，系统在后台持续运行，关键时刻把需要人决策的事情推到前台，并在任务结束后给出可汇报、可审计、可追溯的结果。

05 · 设计机会优先级

下一版界面最值得补强的 6 个点

01

意图确认层

提交意图后先生成结构化摘要，让用户确认目标、区域、指标和周期，降低误解风险。

02

智能体过程透明

借鉴小艺慢思考的反馈方式，用步骤化状态展示系统正在检索、诊断和执行什么。

03

事件优先级解释

接管事件必须标注影响用户、风险等级、置信度、建议动作和不处理后果。

04

地图图层标准化

指标卡与 GIS 图层一一对应，形成健康度、覆盖、质差、节能、投诉的固定视觉语法。

05

人工接管责任边界

明确自动处置、建议处置、禁止自动处置三类动作，避免黑盒自动化带来信任风险。

06

日报证据链

日报不仅说“提升了多少”，还要能回溯到事件、地图位置、策略动作和指标变化。
