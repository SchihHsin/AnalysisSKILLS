Midjourney · tp-02-a/02-b · Stage 2 重新定位

Style Explorer + sref 系统 — 把"风格"提升为可索引、可保存、可混合的一等对象

渲染样本 v1 把这个亮点写成"作品即 prompt"，但"做同款"在即梦/可灵都已普遍存在，这个差异化已被稀释。Midjourney 的真正护城河是 Style Explorer——一个独立的"风格浏览器"，把每个 sref code（视觉风格指纹）作为可点赞、可收藏、可在 prompt 中混用的对象。你可以用 --sref 12345 67890 混合两种风格，再用 --sw 0-1000 调权重——这是把"美学"工程化的方式，超越了"看见就能复刻"的层级。

Style Explorer · 按风格点赞 / 收藏
每个 sref code 是独立可索引对象
混合风格Mix
--sref 12345 67890
--sw 650 (0-1000)
两个 sref + 权重 0-1000，任意组合
可复用Reuse
同一 sref 跨 prompt 保持视觉一致

渲染样本 v1 把这个亮点说成"图形化运镜路径"，但核查后发现 Motion Brush 控制的是画面中物体（人/物）的运动路径，不是镜头运镜。运镜在 Kling 里是另一套——6 种预设方案（推进拉远 / 水平摇镜 / 垂直运镜等）。但这个亮点的核心洞察依然成立："画路径"比"用文字描述运动"更直观——只是把它从"运镜控制"修正为"物体运动控制"。值得抄作业的是这个图形优先于文字的决策，而不是它具体控制了什么。

默认态Idle
未启用运镜控制时为静态构图
绘制中Drawing

应用态Applied
物体沿路径运动 · 0 → 5s

Midjourney · tp-02-b · Stage 2 数据修正

--sref + --sw 系统 — 把"风格"从黑盒提示词变成可量化的旋钮

用文字描述"我想要这张图的色调 + 那张图的构图 + 第三张图的光影"几乎不可能写得精确。Midjourney 的 --sref 让用户把若干张参考图（或 sref code）按权重组合输入，AI 在隐空间里做加权平均。这把"风格"从黑盒提示词变成可视化、可量化的旋钮。

Stage 2 修正 当前 Midjourney 默认是 V8.1（不是 v1 中的 V7），--sw 权重范围是 0-1000（默认 100），不是 v1 mockup 里写的 1-10。来源：MJ 官方 Style Reference 文档

输入Refs
--sref ref1 ref2 ref3
3 张图片或 sref code 作为风格参考
权重调节--sw 0-1000
--sw 100 (默认)
--sw 650 (强)
--sw 25 (弱)
范围 0-1000
权重默认 100，0 弱化 / 1000 强化
输出V8.1
风格按权重融合，可重复可调（V8.1）
